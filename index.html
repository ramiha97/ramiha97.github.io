AI Text Summarization

I developed an AI-driven text summarization and knowledge clustering system designed to efficiently process and condense large textual inputs into concise, context-rich summaries. The system leverages Amazon Bedrock to access advanced LLMs, while incorporating intelligent preprocessing steps—including semantic embedding, clustering, and prompt engineering—to optimize model efficiency and reduce token consumption. Incoming text data is first embedded using Amazon Titan Embeddings, transforming raw text into high-dimensional semantic vectors. A KMeans clustering algorithm groups related segments to ensure thematic coherence, and representative samples from each cluster are selected to form a balanced, information-dense input. The system then dynamically constructs tailored prompts through prompt engineering techniques, guiding the LLM to produce structured, factual, and stylistically consistent summaries. The entire service runs as a microservice within an Amazon EKS cluster, ensuring horizontal scalability and high availability. Each summarization pod runs on optimized CPU-based EC2 nodes, with auto-scaling configured through Kubernetes HPA to handle variable workloads efficiently. A FastAPI (Python) backend orchestrates the end-to-end process—from text ingestion and semantic preprocessing to Bedrock LLM inference and structured output formatting—delivering low-latency, real-time summarization via REST API endpoints.

  • ⚙️ Tools & Techniques AWS Bedrock LLM for large-scale text summarization SentenceTransformers for high-dimensional text embeddings Scikit-learn (K-Means) for clustering and vector reduction NumPy & Pandas for data processing and feature management
  • Tech Stack Language: Python AI/ML Frameworks: AWS Bedrock, Transformers, SentenceTransformers Libraries: Scikit-learn, NumPy, Pandas Architecture: Embedding → Clustering → LLM Summarization Pipeline

Face Recognition System

I developed an AI-powered face recognition system that identifies and retrieves matching faces from large-scale datasets with high accuracy and speed. The system extracts facial embeddings using an InsightFace model deployed on a GPU-enabled EC2 node within an Amazon EKS cluster, ensuring efficient, parallelized inference at scale. All vectorized embeddings are indexed in a Zilliz (Milvus) vector database for high-speed similarity searches, enabling real-time nearest-vector retrieval. Each matched vector is linked to its corresponding image stored in Amazon S3 and enriched with metadata from MongoDB Atlas for context-aware results. The entire pipeline — from inference to retrieval — is containerized and orchestrated via Amazon EKS, leveraging Kubernetes for auto-scaling, high availability, and workload isolation. A FastAPI (Python) backend exposes these capabilities through a unified API layer, delivering seamless, real-time face identification and data retrieval for clients.

  • Tech Stack: Machine Learning & AI: InsightFace, PyTorch Infrastructure: Amazon EKS (GPU Nodes on EC2), Amazon S3 Databases: Zilliz (Milvus) for vector storage, MongoDB Atlas for metadata Backend Framework: FastAPI (Python) Architecture: Microservices orchestrated with Kubernetes, auto-scaling GPU inference pods, and REST API endpoints for client interaction

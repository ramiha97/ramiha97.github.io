AI Text Summarization

Bedrock-powered summarization + semantic clustering for token-efficient, coherent outputs.

Overview

I developed an AI-driven text summarization and knowledge clustering system that condenses large inputs into concise, context-rich summaries. It uses Amazon Bedrock for LLM inference, plus semantic preprocessing (embedding + clustering) to reduce token usage while keeping thematic coherence.

How it works

1) Embed

Incoming text is embedded with Amazon Titan Embeddings into semantic vectors.

2) Cluster

Greedy Cluster groups related segments to keep summaries thematically coherent.

3) Select Representatives

Representative samples are picked from clusters to form a balanced, information-dense prompt input.

4) Summarize

Prompt engineering guides Bedrock LLMs to produce structured, factual, consistent summaries.

Infrastructure

Runs as a microservice inside an Amazon EKS cluster for scalability and availability. Pods run on optimized CPU EC2 nodes. A FastAPI backend orchestrates ingestion → preprocessing → Bedrock inference → formatted output.

Tools & Techniques

  • AWS Bedrock LLM for large-scale summarization
  • Amazon Titan Embeddings
  • Scikit-learn (greedy cluster) for clustering + reduction
  • NumPy & Pandas for processing and feature handling
  • Prompt engineering for structured outputs

Face Recognition System

GPU-accelerated face identification with vector search at scale.

Overview

Developed an AI-powered face recognition system capable of identifying and retrieving matching faces from large-scale datasets with high accuracy and low latency. The system is optimized for real-time inference, scalable retrieval, and metadata-enriched results.

Recognition pipeline

1) Face embedding extraction

Facial embeddings are generated using an InsightFace model running on GPU-enabled EC2 nodes for parallelized inference.

2) Vector indexing & search

All embeddings are indexed in Zilliz / Milvus, enabling high-speed nearest-neighbor similarity searches.

3) Image & metadata retrieval

Matched vectors reference images stored in Amazon S3, enriched with metadata from Postgresql.

4) API delivery

A unified FastAPI backend exposes real-time recognition and retrieval via REST endpoints.

Infrastructure & scaling

The entire pipeline is containerized and deployed on Amazon EKS, leveraging Kubernetes for auto-scaling, high availability, and workload isolation. GPU inference pods scale dynamically based on demand, ensuring consistent performance under load.

Tech stack

Machine Learning

InsightFace, PyTorch

Vector Database

Zilliz / Milvus

Infrastructure

Amazon EKS, EC2 (GPU), S3

Backend

FastAPI (Python)

Outcome

Delivered a production-ready face recognition platform capable of real-time identification, scalable vector search, and metadata-rich results—designed for high-throughput, GPU-accelerated workloads.
